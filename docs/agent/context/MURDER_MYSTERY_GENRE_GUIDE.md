# Murder Mystery Genre Guide

이 문서는 머더미스터리 장르 대화와 시나리오 설계 판단을 위한 보존 노트다.
머더미스터리 게임, 시나리오, 조사, 밀담, 회의, 투표, 엔딩, 기믹을 논할 때는 먼저 이 문서를 읽고 답한다.

이 문서는 장르 문법을 정리한 기준이다. 현재 프로젝트 구현 가능 여부는 [data/murder-mystery/SCENARIO_AUTHORING.md](/c:/Users/xotjr/Desktop/react/next-game-web/data/murder-mystery/SCENARIO_AUTHORING.md)와 실제 소스가 우선한다.

## Core Shape

머더미스터리는 플레이어가 관찰자가 아니라 사건 속 등장인물/용의자가 되어, 자기 비밀과 목적을 가진 채 제한된 정보로 진상을 재구성하는 상호작용형 추리 게임이다.

핵심 긴장은 보통 세 층위에서 생긴다.

- 사건 진상: 누가, 언제, 어디서, 어떻게, 왜 했는가.
- 인물 비밀: 각자 숨기고 싶은 과거, 동기, 알리바이의 구멍, 소지품, 관계.
- 플레이 목적: 공통 목표와 개인 목표가 충돌하거나, 범인은 도망치고 다른 인물은 특정 진실/물건/사람을 지켜야 한다.

좋은 머더미스터리 답변은 "정답 찾기"만 보지 않는다. 정보가 누구에게 있고, 언제 공개되며, 공개하면 어떤 협상/거짓말/의심이 생기는지까지 같이 본다.

## Basic Phase Grammar

작품마다 이름과 순서는 다르지만, 기본 진행은 아래 페이즈들의 조합이다.

| Phase               | 장르상 역할                                                                    | 설계 질문                                                        |
| ------------------- | ------------------------------------------------------------------------------ | ---------------------------------------------------------------- |
| 규칙 안내/역할 배정 | 플레이 범위, 거짓말 가능 여부, 카드 공개 규칙, 스포일러 금지를 정한다.         | 플레이어가 무엇을 해도 되고 하면 안 되는가?                      |
| 사전 지문/개별 시트 | 캐릭터의 기억, 관계, 비밀, 개인 목표, 초기 소지품, 타임라인을 준다.            | 이 인물은 무엇을 알고, 무엇을 숨기고, 무엇을 원하나?             |
| 오프닝/사건 브리핑  | 모두가 공유하는 사건 발생 지점과 최초 질문을 만든다.                           | 첫 회의에서 바로 말할 수 있는 공통 화제가 있는가?                |
| 전체 회의           | 모두가 정보를 공유하고, 알리바이와 추리를 맞추며, 공개 설득을 한다.            | 이 회의 뒤에 의심의 방향이 바뀌는가?                             |
| 밀담                | 2~3명이 따로 협상, 거래, 확인, 거짓말, 비밀 공유를 한다.                       | 전체 앞에서는 말할 수 없지만 특정 상대에게는 말할 이유가 있는가? |
| 조사                | 제한된 횟수/토큰/순서로 장소, 인물, 물건, 문서 등을 조사해 단서 카드를 얻는다. | 조사 선택이 의미 있고, 단서가 새 행동을 만들까?                  |
| 추가 정보/이벤트    | 새 페이지, 봉투, 방송, NPC 증언, 시간 경과, 사건 재발생으로 국면을 전환한다.   | 중반에 플레이어의 기존 가설을 흔드는가?                          |
| 추리 발표/최종 회의 | 각자 최종 이론을 말하고, 방어와 프레이밍을 끝낸다.                             | 투표 전에 플레이어가 결론을 언어화할 시간이 있는가?              |
| 투표/결단           | 범인, 공범, 동기, 진상, 구원/복수 등 최종 판단을 제출한다.                     | 동률, 미제출, 복수 질문, 역할별 추가 선택은 어떻게 처리하나?     |
| 엔딩/엔딩북         | 투표와 선택 결과에 따라 진상, 후일담, 점수, 캐릭터별 결말을 공개한다.          | 플레이어의 선택이 결말 문장에 실제로 반영되는가?                 |
| 감상전/해설         | 모든 비밀, 미공개 단서, 추리 동선을 공유한다.                                  | 플레이 후 억울함보다 납득이 남는가?                              |

현재 프로젝트의 기본 엔진 흐름은 이 중 `intro`, `investigate`, `discuss`, `final_vote`, `ending_choice`, `endbook`에 가깝다.

## Pre-Investigation Text

사용자가 "조사전 지문"이라고 부르는 영역은 보통 아래 내용을 포함한다.

- 공개 도입: 사건 배경, 현재 장소, 피해자 상태, 모두가 아는 공통 정보.
- 개인 도입: 캐릭터의 기억, 직전 행적, 숨겨야 할 정보, 공개해도 되는 정보.
- 관계표: 누구와 어떤 감정/이해관계/빚/협박/연대가 있는가.
- 개인 목표: 범인 찾기 외에 지켜야 할 사람, 숨겨야 할 비밀, 획득해야 할 물건, 특정 투표 결과.
- 행동 제한: 거짓말 가능 범위, 카드 공개 가능 여부, 밀담 가능 여부, 메타 발언 금지.
- 잠금 정보: 특정 라운드 이후 읽는 추가 페이지, 조건부 봉투, 조사 성공 후 읽는 문단.

사전 지문은 많은 정보를 주는 설명서가 아니라, 첫 회의에서 어떤 말을 할지 결정하게 만드는 장치여야 한다.
읽은 직후 플레이어가 "나는 무엇을 숨길까", "누구에게 먼저 물어볼까", "누구를 도와야 하나"를 떠올리면 기능한다.

## Investigation Design

조사는 정보 획득과 선택의 페이즈다.
단서가 그냥 무작위 텍스트 보상이라면 약하고, 이후 회의/밀담/투표에서 행동을 바꾸게 해야 한다.

주요 조사 대상:

- 장소: 현장, 이동 경로, 은신처, 피해자 방, 기록실.
- 인물: 증언, 행동, 관계, 숨긴 물건, 모순된 기억.
- 소지품/문서: 편지, 기사, 의료 기록, 영수증, 사진, 녹음, 디지털 로그.
- 특수 대상: 잠긴 금고, 암호, 시체 재검, NPC, 지도 핫스팟.

단서 유형:

- 확정 사실: 실제로 있었던 사건 조각.
- 증언: 말한 사람의 이해관계 때문에 왜곡될 수 있는 정보.
- 모순 단서: 알리바이, 시간표, 소지품, 상처 위치가 서로 맞지 않게 만든다.
- 동기 단서: "할 수 있었다"보다 "왜 했는가"를 보강한다.
- 미끼 단서: 합리적으로 의심하게 하지만, 더 깊은 단서로 빠져나갈 수 있어야 한다.
- 조건부 단서: 특정 카드 조합, 라운드, 인물, 선택 뒤에 열린다.
- 상태 변화 단서: 이름 공개, 파츠 획득, 새 조사 대상 개방, 엔딩 조건 변화를 만든다.

조사 설계의 기본 안전장치:

- 단 하나의 필수 단서가 사라지면 정답이 불가능한 구조를 피한다.
- 핵심 진상에는 2개 이상의 독립 경로로 닿게 한다.
- 미끼 단서는 틀린 길로 보내되, 탈출 가능한 반증도 같이 둔다.
- 카드 공개/비공개 규칙이 추리 난이도를 크게 바꾸므로 명시한다.
- 개인 목표 때문에 정보를 숨겨도, 전체 게임이 멈추지 않게 한다.

## Knox Fair-Play Gate For Tricks

트릭, 범행 수단, 알리바이, 정체 반전, 은폐 구조를 설계할 때는 녹스의 10계를 현대 머더미스터리 기준으로 반드시 통과시킨다.
원문의 차별적 표현은 그대로 쓰지 않고, `문화적 타자화나 편견을 반전 장치로 쓰지 않는다`는 현대 규칙으로 대체한다.

머더미스터리는 플레이어가 단서를 직접 조사하고 서로 공개/은폐할 수 있으므로, 트릭은 `숨기면 성립하는 정보`가 아니라 `드러나도 여러 해석이 가능한 정보` 위에 세운다.
핵심 단서가 공개될 가능성을 기본값으로 보고, 공개된 뒤에도 범인/관련 인물이 말로 방어하거나 다른 합리적 오독을 만들 수 있어야 한다.

필수 적용:

- 진범, 핵심 관계자, 범행 수단 후보는 초반부터 세계 안에 존재해야 한다. 후반에 처음 등장한 인물, 물건, 장소가 정답이면 안 된다.
- 초자연, 예언, 저주, 기적, 장르 밖 능력은 실제 해답으로 쓰지 않는다. 쓰더라도 초반부터 규칙화하고 자연적 해석 단서를 둔다.
- 비밀 통로, 숨은 방, 이중 구조 같은 공간 트릭은 남발하지 않는다. 쓰면 지도, 동선, 소음, 먼지, 열쇠, 목격 공백 등 조사 단서로 추적 가능해야 한다.
- 특수 독, 전문 장치, 과학 지식, 법의학 지식은 플레이어가 단서만으로 추론할 수 있을 때만 쓴다. 엔딩에서 긴 설명으로 처음 성립시키지 않는다.
- 특정 문화, 외국인, 장애, 정신질환, 소수자성, 직업 편견을 "수상함"이나 범행 가능성의 지름길로 쓰지 않는다.
- 우연, 꿈, 작가의 직감, 탐정의 감으로 트릭이 풀리면 안 된다. 모든 핵심 결론은 조사 카드, 역할 지문, 공개 발화, 물증 중 최소 하나로 역추적되어야 한다.
- 진행자, 해설자, 룰 텍스트, 탐정 포지션의 인물이 플레이어가 검증할 수 없는 권한으로 진실을 숨기면 안 된다. 진범이 플레이어라면 본인의 행동/기억 은폐 규칙도 단서화한다.
- 플레이어가 접근할 수 없는 단서로만 정답을 확정하지 않는다. 엔딩북의 새 정보는 해설이어야지, 정답을 처음 가능하게 만드는 증거이면 안 된다.
- 시점 인물의 필수 생각, 기억, 추리를 작가 편의로 숨기지 않는다. 기억 결손/거짓 기억은 그 자체가 초반부터 드러나는 트릭이어야 한다.
- 쌍둥이, 대역, 변장, 신분 교체, 이름 바꾸기는 초반부터 존재 가능성을 심는다. 얼굴, 목소리, 필체, 습관, 기록 등 반증 가능한 단서를 함께 둔다.

검수 질문:

- 이 트릭의 정답은 플레이어가 이미 받은 단서만으로 엔딩 전에도 말할 수 있는가?
- 핵심 단서가 전체 공개되어도 즉시 정답표가 되지 않고, 회의에서 다툴 여지가 남는가?
- 범인이 숨긴 정보 때문에 추리가 어려운 것인가, 작가가 필요한 정보를 안 줘서 불가능한 것인가?
- 미끼 단서와 반증 단서가 모두 공정하게 배치되어 있는가?

## Meeting And Private Talk

전체 회의는 공개 설득의 무대이고, 밀담은 비공개 협상의 무대다.
둘은 같은 "대화"가 아니라 압력이 다르다.

전체 회의에서 잘 작동하는 것:

- 시간표 정리, 피해자와의 관계 확인, 공개 알리바이 검증.
- 여러 명이 동시에 검증해야 하는 물증 비교.
- 범인에게 방어하거나 다른 사람을 몰아갈 기회를 주는 공개 압박.
- 라운드 종료 직전의 최종 정리와 투표 유도.

밀담에서 잘 작동하는 것:

- 전체 앞에서 말하면 개인 목표가 망가지는 비밀 교환.
- 둘만 아는 관계 확인, 협박, 거래, 카드 교환, 진술 맞추기.
- "너는 나를 의심하지 마라. 대신 이 정보를 주겠다" 같은 협상.
- 범인이 거짓 프레임을 심거나, 제3자가 표를 조정하는 작업.

주의점:

- 필수 진상이 밀담에서만 공유되고 공개 회의로 돌아오지 못하면 나머지 플레이어는 방관자가 된다.
- 밀담이 있으면 별도 공간/채널/시간 제한/가능 인원/카드 공개 규칙을 미리 정해야 한다.
- 후반에는 밀담을 금지하고 전체 회의만 허용하는 작품도 많다. 이러면 최종 설득의 긴장이 올라간다.

## Vote And Ending

투표는 단순 채점이 아니라 플레이어가 지금까지 만든 해석을 세계에 제출하는 장면이다.

투표 방식:

- 단일 범인 지목: 가장 기본적인 형태.
- 복수 질문: 범인, 공범, 동기, 흉기, 장소, 진짜 목적을 각각 묻는다.
- 역할별 질문: 특정 캐릭터에게만 추가 선택지를 준다.
- 동시 공개/비밀 제출: 공개 압박을 줄지, 은밀한 결정을 보장할지 선택한다.
- 다수결/정답제/점수제: 최다 득표가 서사 결과를 만들 수도 있고, 정답 여부가 점수를 만들 수도 있다.
- 동률 처리: 결선 투표, 우선순위, 범인 승리, GM 판정, 특수 카드 발동 중 하나를 정한다.

엔딩 방식:

- 정답/오답 엔딩: 범인을 맞혔는지에 따라 분기.
- 지목 대상별 엔딩: 누구를 범인으로 만들었는지에 따라 후일담이 달라진다.
- 선택 조립형 엔딩: 공통 진상 위에 역할별 선택/아이템/목표 달성 조각을 붙인다.
- 개인 에필로그: 전체 결말과 별도로 각 캐릭터의 비밀, 점수, 후일담을 공개한다.
- 해설 엔딩: 미공개 단서와 정식 추리 동선을 정리한다.

좋은 엔딩은 "정답은 이거였다"만 말하지 않는다.
플레이어가 공개한 사실, 숨긴 비밀, 마지막 투표가 어떤 세계를 만들었는지 보여준다.

## Common Gimmicks

기믹은 특수한 규칙이나 연출이지만, 단순 장식이 아니라 추리/대화/엔딩 중 하나를 바꿔야 한다.

| Gimmick               | 작동 방식                                                    | 주의점                                           |
| --------------------- | ------------------------------------------------------------ | ------------------------------------------------ |
| 범인이 처음엔 모름    | 중반 봉투나 조건부 지문으로 자신이 범인임을 깨닫는다.        | 범인의 초반 행동이 나중에 납득돼야 한다.         |
| 거짓 기억/불완전 기억 | 각자 기억이 일부 틀리거나 빠져 있다.                         | "작가가 속였다"가 아니라 인물 사유가 필요하다.   |
| 정체/이름 공개        | 특정 단서 뒤 역할 표시명이나 관계가 바뀐다.                  | 공개 전후로 질문의 의미가 달라져야 한다.         |
| 조건부 EX 카드        | 특정 조합, 인물, 라운드, 조사 대상에서만 핵심 단서가 열린다. | 못 열어도 대체 추리 경로가 있어야 한다.          |
| 지도/핫스팟 조사      | 장소 선택과 공간 추리가 중요해진다.                          | 좌표가 장식이면 피로만 늘어난다.                 |
| 조사 토큰/차례        | 제한 자원으로 정보 우선순위를 정하게 한다.                   | 선플레이어 유불리를 보정한다.                    |
| 아이템 거래/강탈      | 밀담과 개인 목표를 강화한다.                                 | 핵심 단서 독점 때문에 게임이 멈추지 않게 한다.   |
| 액션 페이즈           | 능력 사용, 조사 방해, 보호, 공개 방송 등으로 상태를 바꾼다.  | 현재 엔진 지원 여부를 먼저 확인한다.             |
| 공개 금지 카드        | 카드 내용을 말로 요약하거나 직접 공개하지 못하게 한다.       | 규칙을 명확히 안 하면 분쟁이 생긴다.             |
| 최종 밀담 금지        | 마지막은 전체 회의만 가능하게 해 공개 설득을 강제한다.       | 전환 타이밍을 분명히 알린다.                     |
| 복수 엔딩 질문        | 범인 외에도 구원/복수/고발/은폐를 묻는다.                    | 선택지가 플레이어 가치관을 실제로 반영해야 한다. |
| 점수와 서사 분리      | 점수는 개인 목표, 엔딩은 서사 결정을 따른다.                 | 점수 승자가 서사상 패배할 수 있음을 안내한다.    |

## Current Project Mapping

이 프로젝트에서 장르 문법을 구현에 매핑할 때는 아래처럼 생각한다.

| Genre Concept       | Current Project Mapping                                                                                             |
| ------------------- | ------------------------------------------------------------------------------------------------------------------- |
| 사전 지문/개별 시트 | `intro`, `roles.publicText`, `roles.secretText`, 프리리드/룰북 화면                                                 |
| 조사                | `investigations.rounds`, `targets`, `cards`, `card.effects`                                                         |
| 전체 회의           | `flow.steps[].kind: discuss`, 단계 설명/공지                                                                        |
| 밀담                | 현재는 별도 `kind`보다 discuss 단계의 운영 규칙으로 표현하는 편이 맞다. 전용 타이머/채널/인원 제한은 엔진 확장이다. |
| 최종 투표           | `finalVote`의 단일 정답 role 모델                                                                                   |
| 투표 뒤 역할별 결단 | `endingChoices`                                                                                                     |
| 엔딩                | `endbook.variants` 또는 `endbook.sections`                                                                          |
| 단서로 상태 변화    | `addPart`, `revealRoleName`, `dynamicDisplayNameRules`                                                              |
| 지도/카드 뒷면      | `investigations.layout.map`, `target.cardBack`, `card.back`                                                         |

현재 엔진이 자연스럽게 지원하지 않는 요구는 시나리오 데이터만으로 해결하려고 하지 않는다.
예를 들어 복수 범인 정답, 라운드당 다회 조사, 실제 탈락/격리, 밀담 전용 방, 카드 거래/강탈, 액션 페이즈 상태 변화, 다중 질문 투표는 상태머신과 스냅샷까지 검토해야 한다.

## Answer Checklist

머더미스터리 관련 대답을 하기 전 확인한다.

1. 지금 질문이 장르 설계인지, 현재 엔진 구현인지 분리했는가?
2. 조사, 회의, 밀담, 투표, 엔딩 중 어떤 페이즈가 바뀌는가?
3. 플레이어가 새로 알게 되는 정보와 새로 할 수 있는 행동이 있는가?
4. 공개 정보, 비공개 정보, 거짓말 가능 범위를 명확히 했는가?
5. 개인 목표가 전체 추리를 막지 않고 긴장만 만들게 했는가?
6. 핵심 진상에 도달하는 경로가 하나뿐이지 않은가?
7. 분기 엔딩이 단순 정답표가 아니라 플레이어 선택의 결과처럼 느껴지는가?
8. 현재 프로젝트 엔진으로 가능한지, 엔진 확장이 필요한지 구분했는가?
9. 트릭, 알리바이, 정체 반전, 범행 수단이 Knox Fair-Play Gate를 통과했는가?
